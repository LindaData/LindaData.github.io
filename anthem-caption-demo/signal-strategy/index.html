How the anthem detector should read any video

This presentation shifts the project from single-clip review into multi-signal software. The detector should combine exact fingerprinting, pitch contour, melody/chroma, tone energy, and later ASR lyric overlap so stadium performances, fan chants, and alternate arrangements can still resolve to the correct anthem.

Build Path

1. FingerprintKeep the current exact-recording pass for high-confidence matches and candidate offsets.
2. Pitch + MelodyExtract frame-level pitch and pitch-class energy so key changes, tempo drift, and singer-led versions still cluster by anthem identity.
3. WordsBatch ASR and lyric keyword overlap become a supporting signal, not the trigger, because sung audio is noisy.
4. Decision LayerScore title prior, fingerprint, melody similarity, and words together, then only replace captions when the combined evidence is strong.
