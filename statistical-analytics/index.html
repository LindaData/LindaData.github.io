Core Workflows

Regression

Start with OLS or GLM

Use transparent baselines for continuous, binary, count, and grouped outcomes before reaching for machine learning.

fit <- lm(TARGET ~ x1 + x2 + x3, data = DATA)
summary(fit)
run_lm_diagnostics(fit, DATA)
Model selection

Tune only after validation is designed

Decision trees, random forests, XGBoost, and regularized models need a clean split before a parameter grid matters.

DATA$split <- make_split(DATA)
train <- DATA[DATA$split == "train", ]
test <- DATA[DATA$split == "test", ]
LLM evaluation

Analyze prompts like experiments

One row per output, claim, rater, or prompt-item pair. Compare score, factuality, confidence, cost, and latency.

summary(lm(score ~ prompt_id, data = LLM_DATA))
mean((LLM_DATA$factual - LLM_DATA$confidence)^2)
SQL

Make model-ready tables

SQL should clarify the grain, feature availability, split assignment, and audit trail before R or Python fits a model.

CREATE OR REPLACE VIEW model_features AS
SELECT target, x1, x2, row_id % 5 AS fold_id
FROM data_table
WHERE target IS NOT NULL;
