Modeling Workflow

The fastest way to teach statistical modeling is to force every project through the same sequence: define the question, identify the outcome structure, inspect the design matrix, diagnose assumptions, fit a baseline, validate it, and explain the result in plain language.

1. Question and Target

Name the decision, the unit of analysis, the outcome, and whether the target is continuous, binary, ordered, categorical, count, rate, duration, spatial, or time-indexed.

2. Data Grain

Make one row mean one thing: person, customer, store-day, patient-visit, county-year, game-team, transaction, or event interval.

3. Diagnostics Before Fit

Profile missingness, outliers, class balance, rare categories, collinearity, leakage, autocorrelation, clustering, and heteroskedasticity risk.

4. Baseline First

Fit a transparent model before a complex one. OLS, logistic regression, Poisson regression, and Kaplan-Meier curves are teaching anchors.

5. Validation

Use train/test splits, time splits, group splits, cross-validation, bootstrap intervals, calibration plots, and sensitivity checks.

6. Communication

Translate coefficients, uncertainty, effect sizes, assumptions, and practical impact. Never stop at p-values or leaderboard metrics.

LLM Analysis from a Statistical Perspective

LLM analysis belongs in this hub because prompt and model behavior can be measured with the same statistical discipline as any other experiment. The unit of analysis should be explicit: output, claim, task, prompt-item pair, evaluator rating, or model-version day.

Prompt Experiments

Randomize items across prompts or use paired item designs. Compare rubric scores, factuality rates, latency, token use, and cost with uncertainty intervals.

Rubric and Agreement

Use structured rubrics, multiple raters, Cohen kappa, ICC, and disagreement review. Never treat a single evaluator as perfect ground truth.

Factuality and Calibration

Estimate hallucination or unsupported-claim rates, model factuality with logistic regression, and compare confidence to observed correctness with Brier scores.

Embeddings and Themes

Use PCA, clustering, and human review to identify recurring failure modes, topic groups, and semantic drift in model outputs.

Production Monitoring

Store prompt version, model version, task mix, tokens, latency, cost, score, factuality, and evaluation timestamp. Drift cannot be diagnosed without versioned data.

Diagnostics Before Modeling

Run these checks before trusting a regression. The scripts in this hub include base R helpers for VIF, condition number, Breusch-Pagan style heteroskedasticity screening, Cook's distance, leverage, missingness, and residual summaries.

Risk What to Check Why It Matters Common Response
CollinearityVIF, condition number, pairwise correlation, aliased coefficients.Coefficients become unstable and standard errors inflate.Remove or combine variables, center interactions, use ridge/PCR, keep theory-driven terms.
HeteroskedasticityResiduals versus fitted values, scale-location plot, Breusch-Pagan or White test.OLS coefficients can stay unbiased, but inference can be wrong.Use robust standard errors, transform target, weighted least squares, or a GLM family.
NonlinearityResidual curves, partial residuals, binned residual plots.A linear term can hide a curved relationship.Add splines, polynomial terms, interactions, or GAMs.
Outliers and influenceCook's distance, leverage, DFBETAs, data entry review.A few rows can dominate coefficients or apparent significance.Validate rows, winsorize only with reason, report sensitivity with and without high-influence points.
MissingnessMissingness rates by variable and segment; missingness indicators.Complete-case analysis can change the population being modeled.Impute carefully, model missingness, add indicators where meaningful, document exclusions.
LeakageFeature timestamp, post-outcome fields, IDs that encode the answer.Validation looks good but production fails.Split by time or group, freeze features at prediction time, remove answer-proxy variables.
Class imbalanceOutcome frequency, confusion matrix, precision-recall, calibration.Accuracy can be misleading when one class dominates.Use class-weighting, threshold analysis, PR AUC, calibration, and cost-based metrics.
OverdispersionResidual deviance divided by degrees of freedom for count GLMs.Poisson underestimates uncertainty when variance exceeds the mean.Use quasi-Poisson, negative binomial, robust SE, or a two-part count model.
Zero inflationObserved zero rate versus model-implied zero rate.A single count process may not fit structural zeros.Use hurdle, zero-inflated, or two-stage models.
AutocorrelationACF, PACF, Durbin-Watson, Ljung-Box, residual runs by time.Time-dependent errors break independent-error assumptions.Add lags, seasonality, ARIMA errors, or cluster by time/entity.
Grouped dependenceRepeated rows by person, store, school, county, team, or cluster.Rows are not independent when groups recur.Use clustered SE, mixed effects, GEE, panel methods, or group-aware validation.
Assumption-specific checksProportional odds, proportional hazards, IIA, stationarity, parallel trends.Each specialized model has a failure mode that changes interpretation.Test the assumption, use a more flexible model, or report sensitivity.

Data Pantry

Start with R built-in datasets because they are stable, small, well documented, and excellent for teaching. Add no-key APIs when the team needs current public data. Use key-based APIs when the value justifies credential management.

Best R built-ins for teaching

mtcars, iris, airquality, ChickWeight, ToothGrowth, PlantGrowth, CO2, Orange, warpbreaks, infert, Titanic, USArrests, state.x77, longley, LifeCycleSavings, Seatbelts, sleep, esoph, VADeaths, Puromycin, and AirPassengers.

Free no-key public APIs to use first

World Bank Indicators API, Open-Meteo Forecast API, and the BLS Public Data API are good starting points. The U.S. Census API is worth using, but its developer page states that API keys are required as of February 2025.

Project Roadmap

These are team-ready project tracks. Each can become a post, Quarto site, RStudio lesson, SQL notebook, or client-facing training module.

Foundation

Regression Bootcamp

Use mtcars, cars, and airquality for OLS, diagnostics, transformations, and interpretation.

Classification

Outcome Probability Lab

Use iris, infert, and Titanic for logistic, multinomial, ordinal, and calibration work.

Counts

Events and Rates

Use warpbreaks and public API event counts for Poisson, negative binomial, offsets, and zero inflation.

Time

Forecasting and Lags

Use AirPassengers, BLS series, and Open-Meteo pulls for ARIMA errors, lags, seasonality, and validation by time.

Groups

Repeated Measures

Use ChickWeight, CO2, and Orange for mixed effects, GEE, and cluster-aware validation.

Causal

Policy Evaluation

Use World Bank or Census panels for fixed effects, difference-in-differences, event studies, and sensitivity analysis.

Survival

Time-to-Event Analytics

Use survival package datasets for churn, reliability, retention, hazard ratios, and censoring explanations.

Spatial

Place-Based Modeling

Use Census or World Bank geography joins for spatial autocorrelation, spatial lag, and local-effect diagnostics.

Bayesian

Uncertainty Lab

Rebuild simple regressions with priors, posterior predictive checks, and partial pooling.

ML

Prediction Benchmark

Compare OLS/logistic baselines to trees, random forests, gradient boosting, and neural networks.

Dimension

PCA and Latent Structure

Use USArrests, survey batteries, and indicators for PCA, PCR, PLS, factor analysis, and SEM.

Reporting

Executive Model Cards

Standardize model cards with target, grain, assumptions, diagnostics, metrics, limitations, and next actions.

Operating Standards

One Model Card Per Analysis

State the target, unit of analysis, predictors, exclusions, assumptions, diagnostics, metrics, and limitations.

No Credentials in Public Sites

Keep keys in .Renviron, environment variables, or a secret manager. Publish metadata and code, not secrets.

Always Keep a Baseline

Every complex model should be compared to a simple baseline and a decision-relevant benchmark.
