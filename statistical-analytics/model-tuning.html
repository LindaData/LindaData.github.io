Model tuning workbench

Tune models like an analyst, not like a leaderboard.

Good tuning is mostly validation design. This page keeps the order clear: baseline first, split second, tuned candidates third, model card last.

Tuning rule

Do not tune before the split is defensible.

Random CV is wrong for future prediction, grouped customers, repeated stores, policy panels, and many LLM evaluations.

Default metric

Regression: RMSE + MAE. Classification: PR AUC + calibration + threshold cost.

The Tuning Sequence

1. Target

Define outcome and prediction moment.

2. Split

Random, time, group, nested, or bootstrap.

3. Baseline

Mean, majority, OLS, logistic, or Poisson.

4. Grid

Small, justified, reproducible search space.

5. Diagnose

Overfit gap, calibration, residuals, stability.

6. Card

Decision record with limits and owner.

Choose The Family

Interpretability

Stepwise, Ridge, Lasso

Use when the formula, coefficients, and feature selection story matter.

Tune: candidate scope, AIC/BIC, lambda, alpha.

Watch: collinearity, unstable selected terms, p-value overclaiming.

Rules

Decision Trees

Use when stakeholders need understandable nonlinear rules.

Tune: depth, minimum leaf size, pruning penalty.

Watch: tiny leaves, unstable splits, overfitting.

Prediction

Random Forests and XGBoost

Use when tabular predictive accuracy matters and interpretation is secondary.

Tune: learning rate, depth, trees, subsampling, regularization.

Watch: leakage, validation overfit, calibration drift.

Copy-Ready Tuning Blocks

R

Stepwise baseline

DATA <- mtcars
TARGET <- "mpg"
PREDICTORS <- c("wt", "hp", "disp", "qsec", "am")

fit <- lm(as.formula(paste(TARGET, "~", paste(PREDICTORS, collapse = " + "))), data = DATA)
step_aic <- step(fit, direction = "both", trace = 0)
summary(step_aic)

    DecisionTreeRegressor(random_state=123),

SQL

Store the search grid

CREATE OR REPLACE TABLE xgboost_grid AS
SELECT *
FROM (
  VALUES
    (0.03, 2, 1, 0.80, 0.80, 1.0),
    (0.10, 3, 5, 0.80, 1.00, 5.0)
) AS t(eta, max_depth, min_child_weight, subsample, colsample_bytree, reg_lambda);

Model Card Standard

If this information is not written down, the model is not ready for review.

Target and grain

What is predicted, when, and what one row represents.

Feature timing

Why every predictor is available before the outcome.

Validation

The split design and why it matches real use.

Candidate results

Baseline and tuned models on the same metrics.

Diagnostics

Overfit gap, calibration, residuals, sensitivity, drift plan.

Limits

Known failure modes, exclusions, and where the model should not be used.
