Model Tuning and Selection Guide

A practical, copy-paste playbook for fine tuning models without losing the statistical discipline: start with a baseline, choose the right validation design, tune only the parameters that matter, check assumptions, and publish a model card with every result.

Use This Order

1. Define target

Name the target, row grain, prediction moment, and whether the goal is explanation, prediction, ranking, or decision support.

2. Profile data

Check missingness, outliers, class balance, leakage, rare categories, target skew, and whether rows repeat inside groups.

3. Pick validation

Use random CV only when rows are independent. Use time splits for future prediction and group splits for repeated entities.

4. Fit baselines

Keep a mean or majority baseline and a transparent model such as OLS, logistic regression, or Poisson regression.

5. Tune candidates

Search a small defensible grid first. Add random or Bayesian search only after the baseline path is working.

6. Diagnose winner

Check residuals, calibration, overfit gap, variable stability, sensitivity to folds, and whether the selected model is interpretable enough.

7. Refit and document

Refit the selected model on the approved training data, score the holdout once, then write a model card.

8. Monitor

Track feature drift, target drift, calibration drift, cost, latency, and data quality after the model is used.

9. Retune only with reason

Retune when data, target, business cost, model family, or deployment population changes.

Python: Grid Search

import pandas as pd
from sklearn.model_selection import GridSearchCV, train_test_split
from sklearn.tree import DecisionTreeRegressor

DATA = pd.read_csv("your_data.csv")
TARGET = "your_target"
PREDICTORS = ["x1", "x2", "x3"]

X_train, X_test, y_train, y_test = train_test_split(
    DATA[PREDICTORS], DATA[TARGET], random_state=123
)

grid = {"max_depth": [2, 3, 5, None], "min_samples_leaf": [1, 3, 5]}
search = GridSearchCV(DecisionTreeRegressor(random_state=123), grid, cv=5,
                      scoring="neg_root_mean_squared_error")
search.fit(X_train, y_train)
print(search.best_params_)

SQL: Feature Table and Folds

CREATE OR REPLACE VIEW model_features AS
SELECT
  id_col,
  target,
  x1,
  x2,
  x1 * x2 AS x1_x2_interaction,
  CASE WHEN row_number() OVER (ORDER BY id_col) % 5 = 0
       THEN 'test' ELSE 'train' END AS split,
  1 + (row_number() OVER (ORDER BY id_col) % 5) AS fold_id
FROM data_table
WHERE target IS NOT NULL;

Write-Up: What Each Tuning Family Is Doing

Family Plain-English Role Parameters That Matter Most Statistical Caution
Stepwise functionsSearch formulas by adding or removing terms using AIC, BIC, or another criterion.Candidate scope, direction, criterion, hierarchy rules.Stepwise selection can be unstable. Validate the selected formula and avoid treating it as causal proof.
Decision treesSplit the data into readable rules that handle nonlinear effects and interactions.Maximum depth, minimum leaf size, minimum split size, pruning penalty.Unpruned trees overfit quickly. Tiny leaves create fragile stories.
Random forestsAverage many trees to reduce variance and improve tabular prediction.Number of trees, variables tried at each split, minimum node size, sampling strategy.Feature importance can be biased. Use group or time validation when rows are not independent.
XGBoost and gradient boostingBuild trees sequentially so each new tree improves on prior errors.Learning rate, tree depth, minimum child weight, subsampling, column sampling, L1/L2 regularization, number of rounds, early stopping.Boosting is powerful enough to exploit leakage. Use a clean validation design and calibration checks.
Regularized GLMsShrink coefficients to handle collinearity, many predictors, and noisy features.Lambda, alpha or l1_ratio, standardization, family/link.Coefficients are biased by design. Interpret shrinkage models as predictive unless the study design supports stronger claims.
SVM and kNNUse margins or local similarity instead of explicit regression coefficients.C, gamma, kernel, k, distance metric, weights.Scaling matters. Performance can collapse with irrelevant features or high dimensionality.
Neural networksApproximate flexible nonlinear patterns when the dataset is large enough.Hidden units, layers, activation, learning rate, regularization, early stopping.They need strong validation and monitoring. They are rarely the first model for small tabular teaching data.
Thresholds and calibrationTurn predicted probabilities into decisions and make probability statements honest.Threshold, cost ratio, calibration method, calibration bins.A high ROC AUC can still have poor calibration or a bad business threshold.
LLM tuningCompare prompts, model IDs, temperature, top-p, rubrics, and evaluators as experimental factors.Prompt version, model version, temperature, top-p, rubric, item assignment.Use paired or randomized designs, measure rater agreement, and compare factuality, cost, latency, and calibration.
Do not tune your way out of a design problem. If the wrong rows are joined, the target leaks into the predictors, the validation split is unrealistic, or the target is not aligned to the decision, a stronger model only makes the wrong answer look more convincing.

Model Card Minimum

Target and Grain

What is predicted, when it is predicted, and what one row represents.

Data and Exclusions

Source tables, date range, filters, missingness handling, and excluded rows.

Validation Design

Random, time, group, nested, or bootstrap validation, with a reason for the choice.

Candidate Results

Baseline and tuned models compared on the same split and metric.

Diagnostics

Collinearity, heteroskedasticity risk, leakage, residuals, calibration, overfit gap, and drift plan.

Decision Rule

How predictions become action: threshold, ranking cutoff, interval, or human review process.

Limits

Known failure modes, weak assumptions, fairness concerns, and where not to use the model.

Owner and Version

Model id, training date, code version, data version, owner, and review date.
